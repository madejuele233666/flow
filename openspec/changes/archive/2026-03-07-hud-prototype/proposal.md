# 提案：沉浸式三阶悬浮 HUD 前端架构 (Hover-to-Interact HUD Prototype)

## 1. 摘要 (Abstract)
当前心流引擎 (Flow Engine) 具备了极其强悍的底层六边形架构、八维状态机以及守护进程。但在展示层，除了雏形版的 TUI 外，缺乏一个能够提供“心流沉浸感”的图形化界面。

本提案旨在提出一套全新的桌面端前端架构：**三阶隐匿式悬浮 HUD (Heads-Up Display)**。该架构通过全局坐标监听与操作系统底层窗口特性的深度结合，实现“静默透明穿透 -> 移回唤醒 -> 悬停实体交互”的极客体验。技术选型拟定为纯 Python 技术栈： `PySide6 (Qt) + Pynput (全局钩子) + 自研 IPC Client`。

## 2. 动机 (Motivation)
- **消除视觉干扰**：传统 GUI 窗口（哪怕是小工具）也会持续抢占屏幕像素与心智，不符合心流“专注无形”的理念。
- **填补交互断层**：TUI 适合黑客式的极少次交互，但要在沉浸与唤醒之间找到平衡点，需要一个能够根据用户行为（鼠标移动/悬停）**智能退让**与**自发觉醒**的浮动组件。
- **高级感 UI 落地**：落地 `aim.md` 中描述的核心特性：“HUD 半透明常驻边缘，默认穿透。仅当悬停达到阈值后实体化”。

## 3. 核心设计：三阶感知模型 (The Three States of Flow)
这是该组件最核心的业务逻辑与动画编排。我们摒弃简单的线性显示隐藏，转而采用与用户注意力曲线高度吻合的物理驱动模型：

### 3.1 幕一：静默态 (The Void / Sentinel Mode)
- **触发条件**：鼠标在全局范围内长时间静止（如 > 3s）或脱离侦测区域。隐喻用户已进入深层心流。
- **UI 呈现**：几乎不可见的 2px 光晕，贴在屏幕边缘。色彩冷峻或随时间流逝而渐渐退去（融入桌面）。
- **动效感观**：带有极缓阻尼感的“抽离”动画，仿佛被屏幕边缘磁吸回去。
- **系统属性**：**100% 开启鼠标穿透 (`WindowTransparentForInput`)**。完全不阻挡下层 IDE/浏览器的点击与操作。不抢夺焦点。

### 3.2 幕二：觉醒态 (The Pulse / Awareness Mode)
- **触发条件**：全局鼠标雷达侦测到鼠标产生位移（打破静止）。隐喻用户心智被激活。
- **UI 呈现**：瞬间拉伸为一条细长的半透明（毛玻璃）状态栏，展示极少信息（如 `[P1] 任务简述 ⏱ 24:15`）。时间字体固定为等宽（Monospace）以防抖动。
- **动效感观**：伴随轻度弹性（Spring）回弹舒展，如果任务带有极高优先级的引力分，其边框会产生微弱的脉冲呼吸晕影。
- **系统属性**：**依然保持鼠标穿透 (`WindowTransparentForInput`)**。此时它仅为视觉投影，鼠标鼠标直接滑过它时绝不产生实体遮挡感。

### 3.3 幕三：交互态 (The Command Center)
- **触发条件**：鼠标处于觉醒态的几何包围盒上方，并且**有意识地悬停超过阈值（如 500ms 交互延迟）**。
- **UI 呈现**：毛玻璃材质从半透转为浑厚的高斯模糊。实体深度的长阴影（Drop Shadow）如拉开幕布般重重投下。完整的控制按钮展现。
- **动效感观**：如同水底浮上水面，从 2D 线条升维为 3D 实体的 Z 轴拔高感。
- **系统属性**：**瞬间解除鼠标穿透**，接管焦点，变回普通的可点击窗体。鼠标一旦脱战，触发逃逸机制并平滑坍缩回静默态。

## 4. 交互美学与材质 (Interaction Aesthetics & Material)
在实现高密度克制表达的同时，引入以下桌面级的高级质感：

1. **真实毛玻璃与透写 (Acrylic / Window Blur)**
   HUD 不能是丑陋的死黑色调。利用操作系统的底层特性（Windows Mica/Acrylic 或 macOS NSVisualEffectView），让 HUD 能够吸纳透写出背后的代码编辑器或壁纸颜色，传递“我在顶层但我是系统一份子”的生态和谐。
2. **基于物理特性的微动画 (Physics-Based Microinteractions)**
   所有的形态流转严禁使用生硬的 `ease-in-out` 线性过渡。采用带有质量和阻尼参数的物理弹簧模型 (Spring Dynamics) 刻画界面的收放。针对按钮的 `Hover` 与 `Click`，必须辅以轻微放大（如 1.05x Scale）以及微妙的光泽变化以提供沉浸反馈。
3. **颜色作为无声的信息载体**
   摒弃繁冗的文字警告。使用 HUD 边框的色温（从专注蓝 -> 白金 -> 警告橙）来进行时间压流提示，利用人类的先验图形认知降低脑力消耗。

## 5. 技术方案选型 (Architecture & Tech Stack)
为实现上述的系统穿透与动画，我们将完全抛却臃肿的 Electron/Web 技术栈，采用系统侵入性更深、性能更强、代码完全统一的 Python 原生方案。

- **UI 渲染核心**：`PySide6` (Qt 6 for Python)。
  - **为什么**：原生支持跨平台的无边框窗体 (`FramelessWindowHint`)、永远置顶 (`WindowStaysOnTopHint`)，以及底层的穿透标志位 (`WA_TransparentForMouseEvents`) 操作。自带高度成熟的底层动画引擎 (`QPropertyAnimation` 和 `QStateMachine`)，足以顺滑支撑我们的弹簧物理特性。
- **上帝之眼（全局钩子）**：`pynput`。
  - **为什么**：Qt 本身的穿透模式会导致其窗口无法监听到由系统分发的 Hover 事件。`pynput` 会在一个独立守护线程中，不依赖 GUI 以极低的系统开销监控绝对坐标。一旦坐标落入预设的 HUD 触发包围盒（Bounding Box）并触发迟滞时间，则向主 UI 线程派发“实体化指令”信号。
- **数据流与 IPC 通信**：`flow_engine.ipc.client.IPCClient`。
  - **为什么**：复用被验证过的异步 JSON-Lines 通信机制。UI 进程只管画出最优雅的空壳与动画。排期、番茄钟、引力算法等大脑算力全部通过 IPC 异步推拉，前后端在进程级物理切分，保障绝对的极简与鲁棒。

## 6. 实施阶段 (Phases)
### Stage 1: 原型与“打破次元壁” (The Physical Hacking)
- 验证最大的技术难点：用 PySide6 和 pynput 实现一个能画图的窗体，以及能在 `穿透` 与 `不穿透` 之间随着鼠标全局悬停而丝滑切换的骨架 Demo。
- *预期产出：一个只显示占位符的色块，能够完美演示“静默退藏”和“凝视实体化”的物理反应，并确保完全不会打断背后的代码输入。*

### Stage 2: 视觉皮层编排 (The Aesthetics)
- 引入高级感样式资源：配置字体层级（数字用等宽），挂载操作系统的底层模糊（Window Blur Hack），精调长阴影（Drop Shadow）参数。
- 注入物理引擎属性：为所有三态的 `QPropertyAnimation` 赋予诸如 `InOutElastic` 一类的非线性缓动特指。增加按钮对鼠标反应的迟滞阻尼效果。
- *预期产出：一套充满高级质感、像有生命力一样随鼠标进退的 UI 组件壳。*

### Stage 3: 中枢神经接驳 (The IPC Binding)
- 初始化 `IPCClient` 异步连接。
- 将后端推送过来的 `TimerTick`（专注时钟跳动）、`TaskStateChanged` 事件绑定到 HUD 的属性观测器里。
- 为界面上的按钮注入触发远程指令（如 `task.pause`, `task.done`）的回调逻辑。
- *预期产出：HUD 完全激活接血，成为代替命令行的主战交互前线。*

## 7. 风险评估 (Risks)
1. **动画性能损耗**：由于我们要高频监控坐标并持续跑微动画，如果在低配老旧电脑上，可能产生卡顿现象。需在代码级避免过多复杂的实时绘图对象。
2. **多屏幕跨端坐标系混乱**：全局 `pynput` 的坐标系在多屏模式下的换比处理极难。由于 Qt 与系统的 DPI 缩放率不同，需进行仔细的屏幕映射换算。
3. **Mac/Windows 操作系统的底层模糊局限性**：Linux 下要实现完美的亚克力模糊和全局钩子较为困难（特别是在 Wayland 发行版上）。若系统级模糊失败或者消耗太大，须具备降级到优雅暗色半透明的设计预案，保证下限不崩塌。

## 8. 开放问题 (Open Questions)
- 美学与效能边界：在 HUD 展开的交互态中，是否需要进一步扩展一个面板来对接我们的 AI 大脑完成 `task_breakdown` 指令交互，还是一切复杂的文字输入仍应回归命令行与 IDE？
- 回答：进一步扩展面板，让用户完全无需依赖命令行，即可完成一切交互。
