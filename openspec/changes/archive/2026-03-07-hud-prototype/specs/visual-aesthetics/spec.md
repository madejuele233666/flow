## 新增需求 (ADDED Requirements)

### 需求：美学模糊材质 (Aesthetic Blur Materials)
HUD 应利用原生操作系统级窗口模糊（如 Windows Mica/Acrylic、macOS NSVisualEffectView）来渲染其背景，允许底层的壁纸或 IDE 透写显示。

#### 场景：背景模糊配置
- **当** HUD 初始化其主窗口组件时
- **则** 它在其透明内容后方应用原生操作系统模糊效果

### 需求：基于物理的动画引擎 (Physics-Based Animation Engine)
HUD 应在所有 UI 过渡动画中采用具有真实质量和阻尼设置的物理弹簧模型，而非线性的淡入/淡出。

#### 场景：从静默态到觉醒态的过渡
- **当** UI 从静默态切换到觉醒态时
- **则** 它以轻微的弹性回弹和缓慢阻尼进行展开

### 需求：色温上下文提示 (Color Temperature Contextual Cues)
HUD 应通过颜色变化静默地传达状态、任务紧迫性和时间压力，而无需繁冗文字。

#### 场景：时间压力颜色转变
- **当** 任务计时器指示接近截止日期的压力时
- **则** 边缘光晕颜色平滑地从冷蓝色过渡到警戒橙色

### 需求：字体层级 (Typography Hierarchy)
HUD 应执行严格的字体区分，计时器和数值数据应排他性地使用等宽字体，以防止快速更新期间出现水平抖动。

#### 场景：计时器渲染
- **当** UI 渲染跳动的倒计时时钟时
- **则** 字符占据固定宽度，避免因数字变化而导致的任何布局重算或抖动
