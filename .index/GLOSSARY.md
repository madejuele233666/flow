# Glossary

- `run`
  - the top-level record for one indexing execution
- `manifest`
  - the snapshot of entries included in a run
- `entry`
  - one stable architectural slice of the repository
- `full_refresh`
  - a repository-wide refresh of the index
- `scoped_refresh`
  - a refresh constrained to declared repository paths
- `boundary_paths`
  - the directories or files an entry is responsible for
- `important_files`
  - high-signal files worth reading first
- `public_interfaces`
  - externally consumed contracts, APIs, CLIs, schemas, or entrypoints
- `validator_status`
  - the run-record summary of contract validation state
