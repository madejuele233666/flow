## 新增需求 (ADDED Requirements)

### 需求：静默态 (Sentinel Mode / Void)
当用户鼠标在指定时间内处于不活动状态或超出探测区域时，HUD 应进入静默且几乎不可见的状态，并允许完全的鼠标输入穿透。

#### 场景：鼠标闲置触发静默态
- **当** 全局鼠标保持静止超过 3 秒时
- **则** HUD 淡化为最小 2px 的边缘光晕，并开启完全的输入穿透 (`WindowTransparentForInput`)

### 需求：觉醒态 (Awareness Mode / Pulse)
当探测到全局鼠标移动时，HUD 应切换为包含极简信息的半透明状态栏，且对鼠标点击保持穿透。

#### 场景：鼠标移动触发觉醒态
- **当** 鼠标在静止后产生移动时
- **则** HUD 展开为半透明状态栏，显示当前任务和计时器，同时保持输入穿透

### 需求：交互态 (Command Center Mode)
当用户有意识地在 HUD 几何包围盒上悬停超过阈值时长时，HUD 应变为完全不透明、不穿透鼠标点击且具有交互性。

#### 场景：有意识悬停触发交互态
- **当** 鼠标在 HUD 包围盒内悬停超过 500 毫秒时
- **则** HUD 切换为带有落阴影的厚重模糊效果，禁用输入穿透，并显示完整的交互控件
