# Past Docs

这个目录存放原始长文、历史文档和已经被 roadmap 吸收后的源材料。

它们保留的目的有三个：

- 保留完整上下文
- 保留历史判断和原始长文
- 允许后续追溯某个路线判断最初来自哪里

但从现在开始，它们**不是现行真相面**。

现行文档入口在：

- [../roadmap/README.md](../roadmap/README.md)

使用规则：

- 需要当前事实、当前路线、当前边界时，看 `docs/roadmap/`
- 需要历史背景、原始长文、旧判断来源时，再看 `docs/past/`

说明：

- 这里的文档不代表全部都“错误”
- 有些仍然有效
- 有些只是历史复盘
- 有些已经部分过时
- 有些是已经拆解吸收的原始愿景文档

包括：

- [aim.md](./aim.md)
  - 原始产品愿景白皮书
  - 其有效部分已经拆入 `docs/roadmap/`

具体时效判断见：

- [../roadmap/08-doc-freshness-audit.md](../roadmap/08-doc-freshness-audit.md)
